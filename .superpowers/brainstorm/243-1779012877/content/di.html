依赖注入方案

ViewModel / Repository / ExoPlayer 实例需要一套管理机制

A

手动注入 + Application 单例

在 Application 类中创建依赖容器，手动将 Repository 传给 ViewModelFactory。
零依赖，完全掌控，但样板代码稍多。

B

Hilt (推荐)

Google 官方 DI，基于 Dagger。对 Service（MediaService）注入支持最好。
编译时验证，但会增加构建时间。

C

Koin

纯 Kotlin DSL，运行时注入。上手简单，无需注解处理器。
但缺少编译时检查，大型项目维护成本上升。

推荐 B — Hilt

因为我们的 AsmrMediaService 是一个 Foreground Service，需要注入两个 ExoPlayer 实例和 Room DAO。
Hilt 对 Android Service 的 @AndroidEntryPoint 支持是最成熟、最稳定的。
